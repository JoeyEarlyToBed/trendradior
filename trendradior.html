01 为什么做

从质量工程师的视角
看舆情监控的痛点

质量问题和公关舆情，本质都是"信息滞后导致的被动救火"。我在库卡的工作中深刻体会到这一点。

⏰

发现滞后 2-3 天

目前公关团队靠人工刷微博、搜百度发现舆情，等内部流程走完，黄金回应期已过。客户投诉在社交媒体发酵，往往比内部工单更早出现。

🔍

信息噪音极大

通用平台热榜充斥着娱乐八卦、营销广告，真正与工业机器人、新能源、医疗器械相关的行业信号被淹没在噪音中。

📝

分析靠"经验直觉"

舆情判断依赖个人经验，缺乏结构化分析。"这条要不要回应？""影响范围多大？"往往没有数据支撑，决策风险高。

💡 核心假设：用开源工具 + AI，24小时能否搭出一个"可用的舆情监控原型"？

04 高管视角

同一个底座
不同高管看不同切片

TOB工业机器人的舆情战场不在大众互联网，而在专业媒体、技术社区、投资研报、供应链传闻。不同高管需要不同的"信息仪表盘"。

CEO / 集团层面

品牌声誉仪表盘

  • "库卡"与"美的"的品牌情绪关联
  • 海外ESG评级与碳中和报道
  • 地缘政治敏感话题（技术外流、数据安全）
  • 资本市场研报对库卡增长预期

核心价值：CEO不一定会看技术参数，但他关心"库卡"在资本市场和行业媒体里的情绪指数，提前预警品牌声誉风险。

CTO / 技术层面

技术竞争雷达

  • 协作机器人 vs 传统工业机器人的路线争议
  • 具身智能/AI+机器人是否颠覆性技术
  • 竞品动态（ABB新技术、发那科扩产）
  • 开源ROS社区对库卡开放性的评价

核心价值：CTO关心"库卡的技术路线在行业对话里有没有失声"——当热词出现时，库卡的声音是多了还是少了。

CDO / 数字化层面

数字化叙事追踪

  • 美的"科技集团"叙事的外部认可度
  • 库卡AI+质量/运维案例的行业报道
  • 数字工厂、灯塔工厂的标杆传播
  • GDPR/数据安全法合规舆情

核心价值：CDO关心"数字化转型故事能不能讲出去、有没有人信"——内部数字化投入需要外部验证。

质量总监 / 业务层面

客户口碑前置信号

  • 客户采购部门的内部评估口碑
  • 招标文件的口碑调查信号
  • 供应链传闻与零部件风险
  • 产品缺陷在客户侧的提前发酵

核心价值：同一个产品缺陷，可能先在客户会议被讨论（舆情信号），然后才变成售后工单（质量数据）。

05 诚实说

效果一般？
是的，但问题很清晰

24小时搭出的原型，能验证技术路径，但离"精准公关决策助手"还有距离。以下是我看到的三个真实瓶颈。

❌ 瓶颈一：数据源太泛，行业信号弱

目前主要抓取知乎、抖音、B站等大众平台，对工业机器人、新能源、医疗器械的垂直行业舆情覆盖不足。真正的行业信号往往在财报电话会、海外论坛、专利诉讼、供应链传闻中，这些需要额外接入垂直RSS源和内部数据。

↓

✅ 解法：接入垂直数据源 + 行业知识库

下一步接入高工机器人、机器之心、IFR国际机器人联合会、36氪产业报道等垂直RSS；同时把库卡历史公关案例、产品技术文档、竞品动态做成知识库，让AI"看懂"专业术语。

❌ 瓶颈二：AI分析浅，不够 actionable

目前的AI输出是"热点概述+情感判断"，对公关总监来说还不够——他需要知道"这条舆情风险等级多高？影响范围多大？建议回应策略是什么？观望/声明/主动沟通？"

↓

✅ 解法：Prompt工程 + 公关决策框架

改写AI分析Prompt，强制输出"风险红绿灯（高/中/低）+ 影响范围预估 + 建议回应策略"。同时接入历史案例库做"相似舆情匹配"，从"开放推断"变成"案例参照"。

❌ 瓶颈三：从"发现"到"行动"是断的

工具目前只到"推送简报"，没有闭环。公关团队收到简报后，还需要手动走内部审批、起草回应、跟踪效果。信息流断了。

↓

✅ 解法：Dify Agent 工作流闭环

用Dify把"舆情发现 → AI根因分析 → 生成回应草案 → 高管审核 → 推送执行 → 效果跟踪"串成Agent工作流。AI做信息整合和初稿，人类做价值判断和最终决策。

06 下一步

从"热榜过滤器"
到"公关决策助手"

目标不是做一个大而全的工具，而是做一个"高管专属的信息仪表盘"——每天只花3分钟，看到最该看的3条。

📋
Phase 1 · Day 1-2

痛点对焦 + 数据摸底

与高管做一次"舆情焦虑访谈"：最近半年最焦虑的舆情事件是什么？从知道到反应花了多久？目前团队每天花多少时间在信息搜集上？用真实痛点确定工具优化方向。

🧠
Phase 2 · Day 3-5

知识库注入 + Prompt 调优

把高管的公关经验结构化：历史上的真实舆情案例、回应策略、效果反馈 → 做成知识库。同时改写AI分析Prompt，让输出从"泛泛概述"变成"风险分级 + 策略建议"。

🔧
Phase 3 · Day 6-7

垂直数据源接入

接入行业垂直RSS源（高工机器人、机器之心、IFR等），替换部分大众平台噪音源。同时接入内部数据源（客户投诉系统、售后工单），实现"内外信息统一看"。

⚡
Phase 4 · Day 8-10

Dify Agent 工作流闭环

用Dify搭建"舆情发现 → AI分析 → 生成回应草案 → 高管审核 → 执行跟踪"的Agent工作流。高管每天只需要花10分钟审AI生成的"待决策清单"。

🎬
Phase 5 · Day 11-14

真实案例验证 + 演示包装

用高管提供的真实历史舆情事件做"回溯验证"：如果当时有这个工具，能提前多久发现？能给出什么不同策略？最后包装成"高管使用前后对比"的演示故事。

粗糙，但已经能跑起来

ChatGPT 第一次也很粗糙，但它让你输入了自己的问题，给了专属答案。

这个舆情工具现在也是粗糙的，但它已经能输入库卡的关键词，给出库卡相关的信号。

给我2周和高管共创，粗糙会变成趁手。
